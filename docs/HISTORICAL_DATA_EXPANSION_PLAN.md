## Angel Source Audit

If the Angel data arrives as a PostgreSQL dump, restore it into a separate database first. Keep it separate from `nse_research_platform` until the audit and staging checks pass.

Example restore:

```powershell
.\scripts\restore_angel_dump.ps1 `
  -DumpPath "D:\path\to\angel_data.dump" `
  -DatabaseName angel_data `
  -UserName postgres `
  -PgBin "C:\Program Files\PostgreSQL\16\bin"
```

Omit `-PgBin` if `psql` and `pg_restore` are already on PATH.

Run the same audit with the Angel source database enabled:

```powershell
$env:ANGEL_DATABASE_URL="postgresql+psycopg://USER:PASSWORD@HOST:PORT/DBNAME"
.\.venv\Scripts\python.exe scripts\run_historical_data_audit.py
```

Optional source table override:

```powershell
.\.venv\Scripts\python.exe scripts\run_historical_data_audit.py --source-table ohlcv_15min
```

The source audit checks:

- Whether the source table exists.
- Distinct source symbols.
- Earliest and latest available 15-minute timestamp.
- Total source rows.
- Current NSE500 symbols missing from the Angel source.

For the current local setup, this will likely look like:

```powershell
$env:ANGEL_DATABASE_URL="postgresql+psycopg2://postgres:YOUR_PASSWORD@localhost:5432/angel_data"
.\.venv\Scripts\python.exe scripts\run_historical_data_audit.py --source-table ohlcv_15min
```

## ETL Sequence

1. Freeze Swing V2.1 logic and expand only OHLCV history.
2. Audit Angel 15-minute coverage by symbol and earliest timestamp.
3. Build a symbol-map review for renamed, merged, and delisted names.
4. Aggregate Angel 15-minute bars to daily OHLCV using NSE session boundaries.
5. Load aggregated bars into a staging table, not directly into `prices_daily`.
6. Compare staged daily bars with existing daily overlaps by symbol/date.
7. Review large discontinuities before recomputing features.
8. Backfill `prices_daily` only after coverage and corporate-action checks pass.
9. Recompute `features_daily`, `sector_daily`, `daily_scores`, and recommendations for the expanded range.
10. Re-run the frozen Swing V2.1 trade and portfolio backtests across the longest validated period.

## Acceptance Criteria

- Earliest validated date reaches at least five years before the re-test end date.
- Source coverage gaps are explained as symbol-map issues, unavailable Angel history, or intentionally excluded securities.
- Corporate-action candidates are reviewed before performance results are trusted.
- Swing V2.1 is re-run without factor, score, or filter changes.
