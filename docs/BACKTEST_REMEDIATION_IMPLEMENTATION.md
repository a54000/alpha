# Backtest Remediation Implementation

**Date:** 2026-06-11

**Objective:** Implement `docs/BACKTEST_REMEDIATION_PLAN.md` by eliminating signal-date close entry from recommendation backtests.

---

## Summary

The recommendation backtest now uses next-trading-day open entry instead of signal-date close entry.

No scoring logic was changed.
No recommendation generation logic was changed.
No factor calculation logic was changed.

The new clean V1 baseline was generated at:

```text
reports/v1_clean_backtest_results.json
```

---

## Code Changes

### 1. Recommendation Backtest Execution

File changed:

- `app/backtesting/run_backtest.py`

Changes:

1. Added `signal_date` to `TradeBacktestResult`.
2. Kept `entry_date`, but changed its meaning to actual execution date.
3. Added `exit_dates` and `exit_prices` to each trade result.
4. Added `next_trading_day_after()` to find the first available trading day after the signal date.
5. Added `execution_window_return()` to calculate:
   - entry at next-trading-day open
   - exit at fixed-horizon close
   - horizon counted from actual entry date
6. Changed price loading from close-only to OHLC:
   - `PricesDaily.open`
   - `PricesDaily.close`
7. Preserved the legacy close-price map as `prices` for compatibility with existing close-to-close helper usage.

Previous behavior:

```text
entry_date = signal_date
entry_price = close(signal_date)
exit_date = signal_date + horizon
exit_price = close(exit_date)
```

New behavior:

```text
signal_date = recommendation_history.date
entry_date = next_trading_day_after(signal_date)
entry_price = open(entry_date)
exit_date = entry_date + horizon
exit_price = close(exit_date)
```

### 2. Benchmark Alignment

File changed:

- `app/backtesting/run_backtest.py`

Changes:

1. Added `matched_window_return()`.
2. Benchmark return now uses each stock trade's actual `entry_date` and horizon-specific `exit_date`.
3. Benchmark entry price uses benchmark open on the stock trade entry date.
4. Benchmark exit price uses benchmark close on the stock trade exit date.
5. Alpha remains calculated only when benchmark data is available.

New benchmark behavior:

```text
benchmark_entry_price = benchmark.open(stock_entry_date)
benchmark_exit_price = benchmark.close(stock_exit_date)
benchmark_return = benchmark_exit_price / benchmark_entry_price - 1
```

### 3. Horizon Calculations

File changed:

- `app/backtesting/run_backtest.py`

Changes:

1. Horizon indexing now starts from the actual entry date.
2. For a 20-day swing horizon:
   - signal is generated after close on `T`
   - entry happens at open on next trading day
   - exit happens at close 20 trading-day steps after entry

### 4. Persisted Backtest Payload

File changed:

- `app/backtesting/run_backtest.py`

Changes:

Persisted `BacktestRuns.config_json["trades"]` now includes:

- `signal_date`
- `entry_date`
- `entry_price`
- `exit_dates`
- `exit_prices`
- horizon returns

### 5. Validation Script Output

File changed:

- `scripts/run_validation_backtest.py`

Change:

The validation script now writes the clean baseline to:

```text
reports/v1_clean_backtest_results.json
```

instead of overwriting the older validation file.

### 6. Tests

File changed:

- `tests/test_backtesting.py`

Changes:

1. Updated expected entry date from signal date to next trading day.
2. Added a gapped-open test that proves the backtest uses next-day open, not signal-date close.
3. Updated persisted payload assertions to check both `signal_date` and `entry_date`.
4. Updated benchmark test expectations for next-day-open entry.

---

## Known Remaining Limitation

This implementation fixes execution look-ahead bias in the recommendation backtest entry assumption.

It does not implement the full `BACKTEST_SPEC.md` portfolio simulator. The backtest still uses fixed-horizon exits and does not yet model:

- stop-loss exits
- target exits
- rank-decay exits
- rebalance calendars
- position sizing
- transaction costs
- slippage

The score bucket analysis limitation documented in `docs/BUCKET_ANALYSIS_AUDIT.md` is also not remediated by this change.

---

## Verdict

The recommendation backtest no longer enters at signal-date close. The new baseline in `reports/v1_clean_backtest_results.json` should replace the prior close-to-close V1 backtest baseline for execution-aware V1 comparison.
